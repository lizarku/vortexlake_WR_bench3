========================================
  웜업 횟수 변경 근거
  작성일: 2026-03-19
  변경: 10회 → 30회
========================================

1. 변경 내용

   main.java warmup() 메서드의 웜업 파일 수를 10개에서 30개로 변경.
   30개 파일 × 약 4,600건/파일 = 약 138,000건의 로그를 처리한 후
   본 벤치마크를 시작한다.


2. 변경 이유: 운영 환경 시뮬레이션

   실제 로그 수집/적재 시스템은 데몬 프로세스로 상시 가동된다.
   서비스 시작 후 수분 내에 JVM JIT 최적화가 완료되고,
   이후 수시간~수개월 동안 웜 상태로 운영된다.

   벤치마크가 운영 환경의 성능을 정확히 반영하려면
   JIT 최적화가 완전히 안정된 웜 상태에서 측정해야 한다.


3. JVM JIT 컴파일 단계와 필요 반복 횟수

   ┌───────────────────┬──────────────┬──────────────────────────┐
   │ JIT 단계           │ 트리거 조건   │ 설명                      │
   ├───────────────────┼──────────────┼──────────────────────────┤
   │ C1 (Client)       │ ~1,500회 호출 │ 기본 최적화                │
   │                   │              │ 인라이닝, 단순 최적화       │
   ├───────────────────┼──────────────┼──────────────────────────┤
   │ C2 (Server)       │ ~10,000회 호출│ 고급 최적화                │
   │                   │              │ 루프 언롤링, 벡터화,       │
   │                   │              │ 이스케이프 분석 등          │
   ├───────────────────┼──────────────┼──────────────────────────┤
   │ OSR (On-Stack)    │ 장시간 루프   │ 실행 중인 루프를           │
   │                   │              │ 최적화된 코드로 교체        │
   ├───────────────────┼──────────────┼──────────────────────────┤
   │ Deopt/Recompile   │ 프로파일 변경 │ C2가 프로파일 기반으로     │
   │                   │              │ 재컴파일하는 과정           │
   └───────────────────┴──────────────┴──────────────────────────┘


4. 파일 수별 JIT 안정화 단계

   ┌────────────┬────────────────┬──────────────────────────────┐
   │ 파일 수     │ 처리 건수       │ JIT 상태                      │
   ├────────────┼────────────────┼──────────────────────────────┤
   │ 10개 (기존) │ ~46,000건      │ C1, C2 트리거 완료            │
   │            │                │ 그러나 비동기 컴파일이         │
   │            │                │ 아직 설치되지 않았을 수 있음    │
   ├────────────┼────────────────┼──────────────────────────────┤
   │ 20개       │ ~92,000건      │ 최적화된 코드 설치 + 안정화    │
   │            │                │ 대부분의 핫스팟 메서드가       │
   │            │                │ C2 컴파일 완료                │
   ├────────────┼────────────────┼──────────────────────────────┤
   │ 30개 (변경) │ ~138,000건     │ Deopt/Recompile 사이클 완료   │
   │            │                │ JIT 완전 안정 상태            │
   │            │                │ 운영 환경의 웜 상태와 동등     │
   ├────────────┼────────────────┼──────────────────────────────┤
   │ 30개 초과   │               │ 추가 효과 미미                │
   │            │                │ 이미 안정 상태 도달           │
   └────────────┴────────────────┴──────────────────────────────┘


5. 왜 30개인가

   (1) C2 트리거(10,000회) + 비동기 컴파일 설치: ~10개 파일
   (2) 최적화 코드 적용 확인 + 안정화: ~10개 파일 추가
   (3) 재컴파일(deopt → recompile) 사이클 완료: ~10개 파일 추가
   합계: 30개 파일

   30개를 넘어서면 JIT 상태 변화가 거의 없으므로
   추가 웜업은 시간 낭비에 해당한다.


6. 공정성 영향

   웜업 횟수 변경은 4개 포맷(tar, kryo, parquet, avro) 모두에
   동일하게 적용되므로 상대 비교의 공정성에는 영향 없음.
   다만 JIT가 더 안정된 상태에서 측정하므로
   각 포맷의 진정한 성능 차이가 더 정확하게 드러난다.
